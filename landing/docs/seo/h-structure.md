# H-Structure — H1/H2/H3 Hierarchy per Blog Post

> **Purpose**: locked-down heading outline for each of the 20 posts. Writers receive this as a brief — they don't re-think structure, they write copy into the skeleton. Primary KW must appear in H1 naturally; secondaries in H2; long-tail variants in H3; PAA questions in FAQ section (which triggers FAQPage schema — see `schema-strategy.md`).
>
> **Rules**:
> - H1 = one per post, contains primary KW exactly or near-exactly. Word count ~50-70 chars.
> - H2 = 5-8 per post. Each H2 ideally carries one secondary KW or answers a PAA question.
> - H3 = 2-4 per H2. Use for long-tail variants, sub-topics, worked examples.
> - FAQ section = 3-5 Q&As at the end. Trigger for FAQPage schema. Questions should mirror PAA.
> - No keyword stuffing. If a section flows better without the KW, drop it — readability > density.
>
> **Target word count**: 1,800-2,500 words per post. Outlines below are heavy on structure to ensure writers don't under-deliver.
