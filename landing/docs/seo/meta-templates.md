# Meta Templates — Titles, Descriptions, OG, Twitter

> **Purpose**: locked-down meta copy for all 20 blog posts. Writers do not invent — they lift from this table. Consistency wins long-term (predictable CTR across posts, brand signature at end of all titles, OG image format uniform).
>
> **Rules**:
> - Meta title: 50-60 chars including " | Procurea". If PKW pushes over, drop trailing " | Procurea" but keep brand in first 60 chars.
> - Meta description: 150-160 chars. Must contain PKW + benefit + soft CTA.
> - OG title: can be more emotional (up to 70 chars, Facebook truncates at ~88).
> - OG description: longer version OK (up to ~200 chars, but 125 is Facebook's sweet spot).
> - Twitter card: `summary_large_image` for all blog posts (1200×630 image minimum).
> - Canonical: self-referencing unless cross-domain.
> - Title case in titles, sentence case in descriptions. US spelling throughout.
>
> **Brand signature**: " | Procurea" — 11 chars inc. space and pipe. Budget 49 chars for the rest.

## Validation Checklist (per post before publish)

1. [ ] Meta title ≤ 60 chars including " | Procurea"
2. [ ] Meta description 140-160 chars, contains PKW
3. [ ] OG title unique vs meta title (avoid duplication penalty)
4. [ ] OG image exists at correct URL, 1200×630 PNG
5. [ ] Twitter card `summary_large_image` (not `summary`)
6. [ ] Canonical URL self-references, no query params
7. [ ] Hreflang tags present (en-us + x-default, PL when available)
8. [ ] No emojis in any meta field (clean SERP appearance)
9. [ ] PKW appears in H1, title, description, first 100 words
10. [ ] Preview with Ahrefs / Portent SERP simulator before publish

---

## Maintenance

- Review meta CTR after 60 days via GSC.
- Posts under 2% CTR on top-10 positions → rewrite description (title is heavier to change once indexed).
- If a post is rewritten substantially, update `dateModified` + resubmit URL to GSC.
- When translating to PL, regenerate meta tags from scratch — do not direct-translate (idioms break).
